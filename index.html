API Documentation

/api/admin

  • GET /: Endpoint for checking.
  • POST /login: Endpoint to login the admin user. Accepts email and password as input.
  • POST /register: Endpoint to register a new admin user. Accepts name, email, and password as input.
  • POST /logout: Endpoint to logout the admin user.
  • POST /activate: Endpoint to activate a student card. Requires admin privileges. Accepts student uid and card id as input.
  • POST /addBalance: Endpoint to add balance to a student account. Requires admin privileges. Accepts uid and amount as input.

/api/vendor

  • GET /: Endpoint for the vendor dashboard.
  • POST /login: Endpoint to login the vendor user. Accepts email and password as input.
  • POST /register: Endpoint to register a new vendor user. Accepts name, email, password, and store name as input.
  • GET /products: Endpoint to get all the products of a vendor. Requires vendor privileges.
  • POST /products/add: Endpoint to add a new product to a vendor's store. Requires vendor privileges. Accepts name, description, price, and image as input.
  • GET /products/:id: Endpoint to get a specific product of a vendor. Requires vendor privileges. Accepts product ID as input.
  • PUT /products/:id/edit: Endpoint to update a specific product of a vendor. Requires vendor privileges. Accepts product ID, name, description, price, and image as input.
  • DELETE /products/:id/delete: Endpoint to delete a specific product of a vendor. Requires vendor privileges. Accepts product ID as input.

/api/student

  • GET /: Endpoint for the student dashboard.
  • POST /login: Endpoint to login the student user. Accepts uid and password as input.
  • POST /register: Endpoint to register a new student user. Accepts name, email, uid, and password as input.
  • POST /logout: Endpoint to logout the student user.

/api/public

  • GET /: Endpoint for the public dashboard
  • POST /products Endpoint to get all the available products.
  • GET /products/:id Endpoint to get a specific product. Accepts product ID as input.
  • GET /vendors Endpoint to get all the available vendors.
  • GET /vendors/:id Endpoint to get a specific vendor. Accepts vendor ID as input.
